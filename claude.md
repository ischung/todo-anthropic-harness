# Claude 에이전트 오케스트레이션 가이드

이 문서는 프로젝트의 전체 개발 루프를 제어하는 하네스 설계도입니다. Claude Code는 다음의 사이클을 준수하여 프로젝트를 진행합니다.

## 1. 워크플로우 루프 (The Loop)

1. **기획 단계 (Planning):** `planner.md` 지침에 따라 `SPEC.md`를 작성합니다.
2. **계약 단계 (Contracting):** Generator와 Evaluator가 이번 스프린트에서 '완성'으로 간주할 테스트 케이스를 합의하여 `SPRINT_CONTRACT.md`를 작성합니다.
3. **구현 단계 (Generation):** `generator.md` 지침에 따라 코드를 작성하고 로컬 서버를 가동합니다.
4. **검증 단계 (Evaluation):** `evaluator.md` 지침에 따라 실제 앱을 테스트하고 `QA_REPORT.md`를 생성합니다.

## 2. 상태 관리 및 핸드오프 (Handoff)

- 각 에이전트는 독립된 세션처럼 행동하며, 오직 **파일(Markdown)**을 통해서만 문맥을 전달합니다.
- 긴 작업 시에는 컨텍스트 누적을 방지하기 위해 각 단계가 끝나면 주요 상태를 파일에 저장하고 대화를 초기화(Reset)하는 것을 권장합니다.

## 3. 완료 및 재수행 기준

- **통과:** `QA_REPORT.md`의 평균 점수가 8점 이상일 경우 다음 기능으로 넘어갑니다.
- **반려:** 점수 미달 시, Evaluator의 피드백을 Generator의 입력값으로 넣어 수정 개발을 지시합니다
