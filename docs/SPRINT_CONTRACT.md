## 1. 이번 스프린트 범위

Sprint 1은 **핵심 CRUD MVP**다. 아래 기능만 구현하며 그 외는 범위 밖(Out of Scope)이다.

| 기능 ID | 설명 | 포함 여부 |
|---------|------|-----------|
| F-01 | 월별 달력 뷰 + 날짜 클릭 시 할일 패널 표시 | ✅ IN |
| F-02 | 할일 생성 (제목, 날짜, 우선순위) | ✅ IN |
| F-03 | 날짜별 할일 목록 조회 | ✅ IN |
| F-04 | 할일 완료 토글 | ✅ IN |
| F-05 | 할일 삭제 | ✅ IN |
| F-06 ~ F-10 | 편집, 뱃지, AI 기능 | ❌ OUT |

## 2. 완료 기준 (Definition of Done)

> **규칙:** 아래 테스트 케이스를 **모두 통과**해야 Sprint 1이 완료된 것으로 간주한다.  
> 단 하나라도 실패하면 Generator에게 반려한다.

---

### TC-01 | 서버 기동 확인

| 항목 | 내용 |
|------|------|
| **목적** | 프론트엔드·백엔드가 정상 실행되는지 확인 |
| **전제 조건** | `npm run dev`, `uvicorn main:app` 실행 |
| **검증 방법** | `GET http://localhost:8000/docs` → 200 OK / `http://localhost:5173` → 화면 렌더링 |
| **합격 기준** | 두 서버 모두 오류 없이 기동, FastAPI Swagger UI 접근 가능 |

---

### TC-02 | 달력 렌더링 및 날짜 선택 (F-01)

| 항목 | 내용 |
|------|------|
| **목적** | 달력이 표시되고 날짜 클릭 시 패널이 전환되는지 확인 |
| **테스트 절차** | 1. 앱 접속 → 달력이 현재 월로 표시됨 확인<br>2. 임의의 날짜 클릭<br>3. 오른쪽 패널에 해당 날짜(예: "2026년 4월 10일")가 표시됨 확인 |
| **합격 기준** | 클릭한 날짜가 패널 헤더에 정확히 반영됨 |

---

### TC-03 | 할일 생성 (F-02)

| 항목 | 내용 |
|------|------|
| **목적** | 새 할일이 DB에 저장되고 화면에 표시되는지 확인 |
| **테스트 절차** | 1. 달력에서 날짜 선택<br>2. 제목 "장보기", 우선순위 "HIGH" 입력 후 저장<br>3. 목록에 "장보기" 항목이 나타남 확인<br>4. `GET /api/todos?date=<선택날짜>` 호출 → 응답에 해당 항목 존재 확인 |
| **합격 기준** | UI 목록 노출 + API 응답에 데이터 존재 (둘 다 충족) |
| **엣지 케이스** | 제목 없이 저장 시도 → 저장되지 않고 경고 표시 |

---

### TC-04 | 날짜별 할일 조회 (F-03)

| 항목 | 내용 |
|------|------|
| **목적** | 날짜가 다른 할일이 섞이지 않고 해당 날짜 것만 표시되는지 확인 |
| **테스트 절차** | 1. A 날짜에 할일 "A 할일" 등록<br>2. B 날짜에 할일 "B 할일" 등록<br>3. 달력에서 A 날짜 클릭 → "A 할일"만 보임<br>4. B 날짜 클릭 → "B 할일"만 보임 |
| **합격 기준** | 각 날짜 패널에 해당 날짜의 할일만 표시됨 |

---

### TC-05 | 할일 완료 토글 (F-04)

| 항목 | 내용 |
|------|------|
| **목적** | 완료 체크 시 시각적으로 구별되고 DB 상태가 변경되는지 확인 |
| **테스트 절차** | 1. 할일 "운동하기" 등록<br>2. 체크박스 클릭 → 완료 처리 (취소선 또는 흐릿하게 표시)<br>3. `GET /api/todos?date=<날짜>` 호출 → `completed: true` 확인<br>4. 다시 체크박스 클릭 → 미완료로 복귀 |
| **합격 기준** | 완료 상태 토글이 UI + DB 양쪽에 반영됨 |

---

### TC-06 | 할일 삭제 (F-05)

| 항목 | 내용 |
|------|------|
| **목적** | 삭제 후 목록에서 즉시 제거되고 DB에서도 사라지는지 확인 |
| **테스트 절차** | 1. 할일 "지울 항목" 등록<br>2. 삭제 버튼 클릭<br>3. 목록에서 사라짐 확인<br>4. `GET /api/todos?date=<날짜>` 호출 → 해당 항목 없음 확인 |
| **합격 기준** | UI 목록 제거 + API 응답에 데이터 없음 (둘 다 충족) |

---

### TC-07 | API 스키마 검증

| 항목 | 내용 |
|------|------|
| **목적** | SPEC에 정의된 API 명세가 실제로 구현되어 있는지 확인 |
| **테스트 절차** | FastAPI Swagger UI(`/docs`) 또는 curl로 아래 엔드포인트 직접 호출 |
| **검증 대상** | `GET /api/todos`, `POST /api/todos`, `PATCH /api/todos/{id}`, `DELETE /api/todos/{id}` |
| **합격 기준** | 4개 엔드포인트 모두 존재하며, POST/PATCH는 올바른 응답 스키마 반환 |

---

### TC-08 | 기본 UX 요건

| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 학습 없이 핵심 기능을 직관적으로 사용할 수 있는지 확인 |
| **검증 항목** | - 할일 추가 UI가 명확히 식별 가능 (버튼 또는 입력창)<br>- 우선순위(HIGH/MEDIUM/LOW)가 색상으로 구분됨<br>- 빈 날짜 선택 시 "할일이 없습니다" 같은 안내 문구 표시 |
| **합격 기준** | 3가지 항목 모두 충족 |

## 3. 채점 기준 (Evaluator 적용)

| 평가 항목 | 배점 | 합격 기준 |
|-----------|------|-----------|
| 디자인 품질 | 25점 | 우선순위 색상, 완료 시각 처리 등 일관성 있음 |
| 기술적 완성도 | 35점 | TC-01~TC-07 모두 통과 |
| 사용성 | 25점 | TC-08 통과, 흐름이 자연스러움 |
| 독창성 | 15점 | 단순 템플릿 이상의 배려 (예: 빈 상태 처리, 로딩 표시 등) |
| **합계** | **100점** | **80점 이상 → 통과 / 미만 → 반려** |

> CLAUDE.md 기준: 평균 8점 이상(=80점 이상)이면 Sprint 2로 진행.

## 4. Out of Scope (이번 스프린트 제외 사항)

- 할일 수정(인라인 편집) → Sprint 2
- 달력 날짜별 뱃지 표시 → Sprint 2
- AI 자연어 파싱 / 주간 요약 → Sprint 3
- 사용자 인증(로그인) → 미정
- 모바일 반응형 → Sprint 2 이후
