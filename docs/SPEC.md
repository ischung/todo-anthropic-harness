> **버전:** 1.0  
> **작성일:** 2026-04-06  
> **작성자:** Planner 에이전트

---

## 1. 프로젝트 개요

### 비전
누구든지 특정 날짜에 할일을 등록하고 관리할 수 있는 **캘린더 기반 Todo 앱**. 로그인 없이 바로 사용 가능하며, AI가 할일 입력을 도와 더 스마트하게 관리할 수 있다.

### 핵심 가치
- **접근성:** 회원가입 없이 즉시 사용
- **날짜 중심:** 달력으로 날짜를 선택해 할일을 등록/조회
- **AI 보조:** 자연어로 할일을 입력하면 AI가 제목·우선순위·태그를 자동 파싱

## 2. 사용자 스토리

| ID | 역할 | 원하는 것 | 이유 |
|----|------|-----------|------|
| US-01 | 사용자 | 달력에서 날짜를 클릭해 그날의 할일 목록을 본다 | 날짜별로 계획을 파악하고 싶어서 |
| US-02 | 사용자 | 새 할일을 제목·날짜·우선순위와 함께 등록한다 | 할일을 빠뜨리지 않기 위해 |
| US-03 | 사용자 | 할일을 완료 표시하거나 삭제한다 | 진행 상황을 추적하기 위해 |
| US-04 | 사용자 | 할일 내용을 수정한다 | 계획이 바뀌었을 때 업데이트하기 위해 |
| US-05 | 사용자 | 자연어로 "내일 오후 3시 팀 미팅"처럼 입력하면 AI가 날짜·제목을 자동 파싱해준다 | 빠르게 입력하기 위해 |
| US-06 | 사용자 | 달력에서 할일이 있는 날짜에 뱃지(개수)가 표시된다 | 바쁜 날을 한눈에 파악하기 위해 |

## 4. 데이터 모델

### Todo
```
Todo {
  id          : string (UUID)
  title       : string (필수, 최대 200자)
  date        : string (YYYY-MM-DD, 필수)
  priority    : enum ["LOW", "MEDIUM", "HIGH"] (기본: MEDIUM)
  completed   : boolean (기본: false)
  created_at  : datetime
  updated_at  : datetime
}
```

## 5. API 명세

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/todos` | 전체 조회 (query: `?date=YYYY-MM-DD`) |
| POST | `/api/todos` | 새 할일 생성 |
| PATCH | `/api/todos/{id}` | 할일 수정 (부분 업데이트) |
| DELETE | `/api/todos/{id}` | 할일 삭제 |
| POST | `/api/ai/parse` | 자연어 → 할일 데이터 파싱 (AI) |
| GET | `/api/todos/summary` | 이번 주 할일 AI 요약 |

## 6. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | React 18 + Vite | 컴포넌트 기반, 교육적 명확성 |
| UI | Tailwind CSS | 유틸리티 클래스로 빠른 프로토타입 |
| 달력 | react-calendar (또는 직접 구현) | 경량, 커스터마이즈 용이 |
| Backend | FastAPI (Python) | 자동 API 문서, 빠른 개발 |
| DB | SQLite + SQLAlchemy | 설치 불필요, 구조 파악 쉬움 |
| AI | Anthropic Claude API | 자연어 파싱, 요약 기능 |

## 7. UI/UX 전략

### 레이아웃 (2-Panel)
```
┌─────────────────────────────────────────────────────┐
│  📅 DateTodo                              2026년 4월  │
├──────────────────────┬──────────────────────────────┤
│                      │  📋 2026-04-06 할일           │
│   [달력 컴포넌트]    │  ─────────────────────────── │
│                      │  ☐ [HIGH] 팀 스프린트 회의   │
│   날짜에 뱃지(숫자)  │  ☑ [LOW]  이메일 확인        │
│   표시               │  ─────────────────────────── │
│                      │  + 새 할일 추가...            │
│                      │  ✨ AI로 빠르게 입력          │
└──────────────────────┴──────────────────────────────┘
```

### 디자인 원칙
- **우선순위 색상:** HIGH=빨강, MEDIUM=노랑, LOW=초록
- **완료 항목:** 취소선 + 흐릿하게 표시
- **AI 입력창:** 일반 입력창 아래 별도 토글 영역

## 9. 제약 사항 및 설계 원칙

1. **단순성 우선:** 인증(Auth) 없이 시작. 향후 필요 시 추가.
2. **CORS 허용:** 개발 편의를 위해 `localhost:5173` ↔ `localhost:8000` CORS 설정 필수.
3. **환경 변수:** `ANTHROPIC_API_KEY`는 반드시 `.env` 파일로 관리 (하드코딩 금지).
4. **에러 처리:** API 실패 시 사용자에게 toast 알림으로 안내.
5. **반응형:** 최소 768px 이상 데스크탑 기준 레이아웃.
